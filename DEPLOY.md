# GRACE HITL annotator — local hosting

This repo IS the static site plus the helper scripts. It is hosted **from this
machine** with `serve.py` (no build step, no dependencies). The git remote points
at https://github.com/johnsk95/grace_hil.git for backup/version control.

## 1. Serve it
```bash
python serve.py            # default port 8000
# or: python serve.py 9000
```
On startup it prints:
- `http://localhost:8000/` — on this machine
- `http://<lan-ip>:8000/` — for validators on the same network (share this)

Leave it running while validators annotate. Ctrl-C to stop.

## 2. Reaching validators off your network
`serve.py` is only reachable on your LAN. For remote validators, expose it with a
tunnel — this also gives an **https** URL (a secure context, so `crypto.randomUUID`
is used for validator ids; on plain http a still-unique fallback id is used).

- **cloudflared** (no signup): `cloudflared tunnel --url http://localhost:8000`
- **ngrok**: `ngrok http 8000`

Share the tunnel's https URL. Keep both `serve.py` and the tunnel running.

## 3. Wire up label collection (Google Apps Script)
Collection is cloud-based and works the same whether the page is served locally,
via a tunnel, or from Pages.
1. Create a Google Sheet → Extensions → Apps Script → paste `apps_script/Code.gs`.
2. Run `setup` once (approve permissions).
3. Deploy → New deployment → **Web app**: *Execute as: Me*, *Who has access: Anyone*.
4. Put the `…/exec` URL into `config.js` → `APPS_SCRIPT_URL`. Restart `serve.py`
   is not needed (it re-reads files per request), but re-load the page.
   Re-deploy a new Apps Script version whenever you edit `Code.gs`.
5. Set `N_VALIDATORS_EXPECTED` in `config.js` to the number of validators you recruit.

Until `APPS_SCRIPT_URL` is set the site still works, but labels are download-only
(button, bottom-left) and every validator sees the whole pool.

## 4. Push to GitHub (backup / version control)
```bash
git push -u origin main
```
(The repo is already initialized with a first commit and the remote set.)

## Updating the claim pool
`pool.json` here is the current QuALITY pool (210 claims, oracle labels stripped),
generated by `export_pool.py` on this machine so oracle labels never get published:
```bash
python export_pool.py --pool <path-to>/annotation_pool.jsonl --out pool.json
# for the QASPER study instead, from the project root:
python hitl_qasper/run_hitl_qasper.py export-pool && cp ../hitl_web/docs/pool.json pool.json
```
Re-loading the page picks up the new pool (served with no-store).

## Collecting results
```bash
# Google Sheet → File → Download → CSV, then:
python sheet_to_labels.py --csv labels_sheet.csv --out <output_dir>/labels.jsonl
# then run replay/qa/metrics from the harness.
```
